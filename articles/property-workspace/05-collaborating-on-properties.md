## Help Content

**Title:** Collaborating on Properties
**Format:** Help article
**Audience:** All
**Slug:** collaborating-on-properties
**Purpose:** Work together with team members and partners on shared properties.

**Content:**

Collaboration on ReiSearch happens through shared access. When you share a property, other users can see it, discuss it, and contribute based on their permission level.

### Sharing a Property for Collaboration

1. Open the property.
2. Click "Share."
3. Select recipients: individual connections, your team, or specific users by email.
4. Set the permission level.
5. Send.

Recipients receive a notification and the property appears in their Shared tab.

### Viewing Shared Properties

Properties shared with you appear under Properties > Shared. Each card shows the property photo, details, and who shared it.

Click any shared property to open it. The actions you can take depend on your permission level (View, Edit, or Admin).

### Permission Levels on Shared Properties

| Level | Can View | Can Edit | Can Share | Can Delete |
|-------|----------|----------|-----------|------------|
| View | Yes | No | No | No |
| Edit | Yes | Yes | No | No |
| Admin | Yes | Yes | Yes | Yes |

The property owner can always change permissions or revoke access.

### Collaborating on a Property

With edit or admin access, you can:

- Update property details (price, beds, baths, description).
- Add or remove documents and photos.
- Run comps and save them to the property.
- Generate and share presentations.
- Add the property to your folders.

Changes made by collaborators are visible to everyone with access to the property. There is no version history for individual field changes.

### Communication on Shared Properties

Use property threads to discuss the deal with collaborators:

- Open the property and go to the Threads tab.
- Start a new thread or reply to an existing one.
- @mention specific users to notify them.
- Attach files to messages.

Threads keep all deal-related communication attached to the property instead of scattered across email or messages.

### Revoking Access

If a collaborator no longer needs access:

1. Open the property.
2. Go to the sharing or access section.
3. Find the user or team.
4. Click "Remove Access."

The property disappears from their Shared tab immediately. Their existing threads and comments remain but they cannot view the property or participate in new threads.

### Leaving a Shared Property

If someone shared a property with you and you no longer want it in your list:

1. Open the property.
2. Go to the property options.
3. Select "Remove from My View" or "Leave Property."

This removes it from your Shared tab only. The property owner is not notified.

### Version History

Currently, ReiSearch does not maintain a field-level version history for property edits. Significant changes (status moves, document uploads, thread messages) are timestamped. For change tracking, communicate major updates through property threads.
